# RestaurantBook - reservation service

Веб-приложение для автоматизации бронирования столиков в ресторане.

## Стек

- Frontend: React + Vite
- Backend: Django + Django REST Framework
- Database: PostgreSQL
- Runtime: Docker + docker-compose

## Запуск

1. Скопировать настройки окружения:

```bash
cp .env.example .env
```

2. Запустить проект:

```bash
docker compose up --build
```

3. Открыть приложение:

- Frontend: http://localhost:5173
- Backend API: http://localhost:8000/api/
- Django admin: http://localhost:8000/admin/

## Демо-доступы

После запуска автоматически создаются пользователи:

- Администратор: `admin` / `admin12345`
- Пользователь: `user` / `user12345`

## Основные возможности

- регистрация и вход пользователей;
- просмотр меню ресторана;
- проверка доступности столика;
- визуальный предпросмотр выбранного столика при бронировании;
- создание бронирования;
- внутренние уведомления через значок колокольчика;
- email-уведомления в консольном режиме;
- PDF-подтверждение бронирования;
- административное управление бронированиями;
- CRUD столиков и меню;
- экспорт отчётов CSV/PDF.

## Работа с изображениями

В административном разделе реализована загрузка изображений:

- `Админ -> Столики`: можно добавить или заменить изображение столика. Это изображение показывается пользователю на странице бронирования справа после выбора номера столика.
- `Админ -> Меню`: можно добавить или заменить фото блюда. Фото отображается на странице меню ресторана.

Если для столика изображение не загружено, приложение показывает стандартную схему столика из `frontend/public/table-images/`.

## Тестирование

Автоматические тесты backend проверяют регистрацию и вход, создание бронирования, проверку доступности столиков, запрет двойного бронирования, права администратора, загрузку изображений столиков и блюд меню, а также экспорт отчётов.

После запуска контейнеров тесты можно выполнить командой:

```bash
docker compose exec backend python manage.py test
```

Для полного пересоздания базы перед проверкой можно выполнить:

```bash
docker compose down -v
docker compose up --build
```
